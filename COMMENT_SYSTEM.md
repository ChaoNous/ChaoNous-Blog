# 评论系统运维说明

## 文件位置

- 站点评论组件：`E:\ChaoNous-Blog\src\components\comment\Cnc.astro`
- 文章页接入点：`E:\ChaoNous-Blog\src\pages\posts\[...slug].astro`
- 评论配置入口：`E:\ChaoNous-Blog\src\config.ts`
- 已关闭的本地后台占位页：`E:\ChaoNous-Blog\public\admin\index.html`

## 当前接入方式

当前站点使用外部评论服务接入，不在本仓库内自建评论后端。

- 评论 API 域名：`https://cwd-api.198665x.workers.dev`
- 前端 widget 版本：`0.1.11`
- 前端 widget 加载地址：`https://unpkg.com/cwd-widget@0.1.11/dist/cwd.js`
- 评论语言：`zh-CN`
- 文章页启用方式：仅文章详情页挂载评论区

## 评论线程主键策略

当前统一使用文章 `canonicalUrl` 作为评论线程标识。

- `postSlug`：传入文章 canonical URL
- `postUrl`：传入文章 canonical URL
- 不再依赖当前浏览器地址栏路径作为唯一线程主键
- 这样可以避免 alias、尾斜杠差异、访问路径差异导致同一篇文章拆成多个评论线程

## 当前运行行为

评论区的加载流程如下：

1. Astro 在文章页渲染评论容器。
2. 当评论区接近视口时，前端懒加载外部 widget。
3. widget 向外部评论 API 拉取评论、点赞状态和访问统计。
4. 若脚本加载失败，页面显示可见的降级提示，并允许用户手动重试。

## 失败降级策略

当前已启用显式降级提示：

- 默认占位文案：`评论区正在加载…`
- 加载失败文案：`评论系统暂时不可用，请稍后再试。`
- 用户可点击 `重试加载` 再次请求 widget

说明：

- 即使评论系统异常，文章正文仍可正常访问
- 评论失败不会阻断整页渲染

## 后台管理状态

当前仓库内的 `/admin` 页面不是可用后台，而是安全下线提示页。

- 本地后台入口：`https://chaonous.com/admin/`
- 当前状态：关闭
- 关闭原因：不再接受静态内嵌管理密码或静态令牌
- 重新开放前提：接入受保护的认证方案

## 审核与运营建议

当前仓库不直接保存评论审核逻辑，审核能力依赖外部评论服务配置。

建议长期维护时至少记录以下信息：

- 评论是否开启人工审核
- 管理后台实际访问地址
- 管理员认证方式
- 通知方式是否启用
- 是否开启评论点赞与文章点赞

如果以上信息发生变化，应同步更新本文件。

## 迁移策略

若后续需要替换评论服务或迁移到自建后端，建议按以下顺序执行：

1. 保留 `canonicalUrl` 作为稳定线程主键。
2. 先确认旧系统是否支持导出评论与回复层级数据。
3. 新系统导入时保持文章 URL 主键不变。
4. 先在预发布环境验证历史评论映射是否正确。
5. 确认点赞、审核状态、访客身份记忆等能力是否需要迁移。
6. 完成迁移后，再切换 `src/config.ts` 中的评论服务配置。

## 维护注意事项

- 不要把 widget 版本重新改回 `latest`
- 不要让 alias URL、无尾斜杠 URL、临时访问地址混入评论主键
- 不要在仓库中写入评论后台令牌或管理员密钥
- 若评论服务域名、widget 版本或后台策略变化，必须同步更新本文件
